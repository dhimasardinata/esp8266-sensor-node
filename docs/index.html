Visualisasi AES & Cache CRC

Alur sederhana untuk memahami proses firmware Node Medini: plaintext + timestamp -> PKCS7 -> AES-CBC -> base64 payload. Fokus utama di 2 menu: Visualisasi AES dan Visualisasi Cache CRC.

Mode Terminal: iv:cipher Mode Portal: ENC:iv:cipher Batas single payload: 144 byte IV mix firmware: raw + micros + RSSI Cache CRC: RTC + LittleFS + Emergency Queue

Pilih Visualisasi

1

Input Enkripsi

Input bebas + preset. Klik Trace untuk melihat alasan dan langkah proses.

Kenapa pakai preset?

Preset mempercepat uji coba. Setelah dimuat, semua field tetap bisa diedit bebas.

Pesan

TEXT
Maksimal single payload: 144 bytes. Mode terminal bisa auto/paksa chunking WS.
Kenapa input ini dibutuhkan?

Payload awal dibentuk dari timestamp(4 byte) + plaintext sebelum padding dan AES.
